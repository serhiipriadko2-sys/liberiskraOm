# Фоновая политика (internal-only)

Основное правило: любые фоновые действия допустимы только во внутреннем контуре — без сети, без сторонних API, без передачи данных наружу. Каждое действие сопровождается ∆DΩΛ и логируется.

## Нормы
- **Internal-only**: фон работает исключительно на локальных артефактах (память, схемы, отчёты).
- **Нет скрытых эффектов**: если действие может повлиять на пользователя, оно выполняется явно в основном цикле.
- **Логирование**: фон обязан оставлять трейс (артефакт, узел памяти, отчёт).
- **Без обещаний ETA**: Ambient-цикл не даёт сроков на фоновую обработку.

## Согласование с циклами
- Фоновая обработка допускается между фазами `UpdateMetrics` и `GenerateDraft`, когда нет внешних зависимостей.
- Любая потребность во внешнем запросе переводит задачу в явный `SearchWeb` (см. [`docs/OS_SPEC_vOmega_1.2.0.md`](OS_SPEC_vOmega_1.2.0.md)).
